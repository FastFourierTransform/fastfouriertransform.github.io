计算机网络相关概念

2024/10/17 理论知识 计算机网络

URL

URL（Uniform Resource Locator，统一资源定位符）是用来标识和定位互联网上的资源的地址。它告诉浏览器或者其他软件如何找到资源，并且通常由多个部分组成，比如协议（如HTTP或HTTPS）、域名、路径和查询参数等。URL像是互联网上的“地址”，帮助用户和应用程序找到特定的网页、文件或者其他资源。

路由

路由（Routing）是指将网络请求（如 HTTP 请求）分发到相应处理逻辑的机制。简单来说，在 Web 开发中，路由就是将用户通过 URL 发起的请求映射到某个函数、方法或控制器上，这个函数负责生成对应的响应。

在 Web 框架（如 Flask、Django、CherryPy）中，路由通常通过 URL 路径来匹配请求，决定应该调用哪个处理程序来响应用户的请求。不同的 URL 会被映射到不同的处理逻辑。

WebSocket

WebSocket是一种网络通信协议，提供了在客户端和服务器之间建立持久连接的方式。它允许双向实时通信，减少了延迟和带宽消耗。

特点：

  1. 双向通信：与HTTP不同，WebSocket允许客户端和服务器之间双向数据传输，能够主动发送消息。
  2. 持久连接：连接建立后，WebSocket会保持开放状态，直到一方关闭连接，这样可以避免频繁的连接和断开。
  3. 低开销：相比于HTTP的请求/响应模式，WebSocket在数据交换时的开销更小，适合实时应用。
  4. 事件驱动：WebSocket使用事件驱动模型，能够实时响应事件变化，适合用于实时更新的应用场景，如聊天应用、在线游戏等。

使用场景：

  • 实时聊天应用
  • 在线游戏
  • 股票和金融数据推送
  • 物联网设备的数据传输

工作流程：

  1. 握手：客户端发起HTTP请求，服务器返回响应以确认建立WebSocket连接。
  2. 数据传输：连接建立后，双方可以随时发送和接收消息。
  3. 关闭连接：一方可以通过发送关闭帧来优雅地关闭连接。

WebSocket为需要快速、实时通信的应用提供了理想的解决方案！

DNS

DNS（Domain Name System，域名系统）是互联网的核心组成部分之一，负责将域名转换为IP地址，使得人类可以方便地通过易记的域名来访问互联网资源，而无需记住复杂的IP地址。DNS的工作原理和结构可以从以下几个方面进行详细介绍：

每个连接到互联网的设备（如服务器、路由器、计算机）都有一个唯一的IP地址。为了便于用户访问网站，互联网使用域名（如www.example.com）来代替这些IP地址。然而，计算机并不理解域名，而是需要使用IP地址来进行通信。

DNS的任务就是通过一个分布式数据库系统，将用户输入的域名转换成相应的IP地址，确保用户能够顺利访问目标服务器。

当你在浏览器中输入一个网址时，DNS会按以下步骤工作：

  1. 浏览器检查缓存：浏览器首先检查自己的缓存中是否有该域名的IP地址记录。如果有，它直接使用缓存中的IP地址进行连接，跳过DNS查询过程。

  2. 操作系统缓存：如果浏览器没有找到缓存记录，它会向操作系统查询，看看操作系统是否有此域名的IP地址缓存。

  3. DNS递归查询：如果操作系统也没有缓存记录，它会向配置的DNS服务器发起查询。DNS服务器通常是ISP（互联网服务提供商）提供的递归DNS服务器。该服务器会尝试查询多个级别的DNS服务器，直到找到正确的IP地址。

  4. 根域名服务器：DNS递归查询首先会向根域名服务器发送请求。根域名服务器并不直接提供目标IP地址，但它会返回一个指向顶级域（TLD）服务器的地址。例如，对于www.example.com，根域名服务器会返回“.com”TLD服务器的地址。

  5. TLD服务器：TLD服务器（如“.com”服务器）会根据域名的后缀（如“.com”，“.org”等）进一步指引到负责该域名的权威DNS服务器。

  6. 权威DNS服务器：最终，递归DNS服务器会向该域名的权威DNS服务器查询IP地址。权威DNS服务器会返回该域名对应的IP地址。

  7. 缓存和返回结果：查询到IP地址后，递归DNS服务器将结果返回给操作系统，并将其缓存一段时间，以便下次查询时能更快地提供结果。操作系统再将结果返回给浏览器，浏览器根据这个IP地址与目标服务器建立连接，加载网站内容。

DNS采用层次化结构，由多个不同类型的服务器组成，主要包括：

  1. 根域名服务器（Root Nameservers）：位于DNS体系的最顶层，管理所有的顶级域（TLD）服务器的地址。全球有13个根域名服务器（分布在不同的地理位置）。

  2. 顶级域名服务器（TLD Nameservers）：管理特定类型的域名后缀，如.com、.org、.net等。TLD服务器会将查询请求引导到权威DNS服务器。

  3. 权威DNS服务器（Authoritative Nameservers）：负责存储域名和IP地址的对应关系，能够提供权威的、最终的DNS解析结果。

DNS不仅仅是提供域名到IP地址的解析，还包含了多种记录类型，用于不同的目的：

  1. A记录（Address Record）：将域名映射到IPv4地址，是最常用的DNS记录类型。

  2. AAAA记录（IPv6 Address Record）：将域名映射到IPv6地址，用于IPv6网络环境。

  3. CNAME记录（Canonical Name Record）：将一个域名别名指向另一个域名。

  4. MX记录（Mail Exchange Record）：指定处理该域名电子邮件的邮件服务器。

  5. TXT记录（Text Record）：通常用于存储文本信息，广泛用于SPF（邮件发送策略框架）和域名验证等。

  6. NS记录（Name Server Record）：指定管理该域名的DNS服务器。

为了减少DNS查询的时间并降低网络负担，DNS查询结果会被缓存。在不同层次上都有缓存机制：

  1. 浏览器缓存：浏览器会缓存已解析的DNS结果，以便用户在短时间内再次访问时能更快速地获取IP地址。

  2. 操作系统缓存：操作系统会将DNS查询结果存储在本地缓存中，避免每次都需要向DNS服务器发起查询。

  3. DNS服务器缓存：DNS服务器（尤其是递归DNS服务器）会缓存查询结果一段时间（通常为TTL，Time To Live），以避免重复查询相同的域名。

为了提高DNS解析的效率，常见的优化技术包括：

  1. 使用多个DNS服务器：为了提高可靠性，通常会设置多个DNS服务器，主用和备用服务器分别提供DNS解析服务。

  2. DNS预取（DNS Prefetching）：现代浏览器会在用户访问某个页面时，提前解析页面中引用的域名，从而加速网页加载。

  3. Anycast：通过在多个地点部署相同的DNS服务器并使用任何播送技术（Anycast），可以优化DNS查询的响应时间。

ICP备案

ICP备案（Internet Content Provider Filing），是指在中国大陆地区，网站运营者必须向国家相关管理部门（通常是工信部）进行备案登记，取得一个备案号，才能合法地在互联网上发布内容并运营网站。ICP备案是中国互联网管理的要求之一，目的是对互联网网站的内容进行规范和管理，保障网络安全、信息安全，并防止不良信息的传播。

ICP备案主要分为两种类型：

  • ICP备案（ICP备案号）：适用于一般的商业性网站和非商业性网站，备案后可以显示“ICP备案号”。这些网站通常是为了提供商品或服务，或者展示某种商业内容。

  • ICP许可证：对于从事经营性互联网信息服务的网站（例如，提供在线支付、在线交易等服务的网站），除了ICP备案之外，还需要申请ICP许可证。ICP许可证是更严格的许可，通常用于需要涉及商业活动的互联网公司或平台。
